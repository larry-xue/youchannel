Postmortem

What we built, why we built it, and why we stopped.

A candid technical write-up for an open-source AI language learning experiment built around YouTube, Gemini, Supabase, and live speaking practice.

YouChannel was an open-source product experiment: take YouTube videos that learners already care about, analyze them with AI, and turn them into interactive language practice. It was not a landing-page concept. It had a TanStack Start app, Supabase schema, YouTube OAuth, Gemini analysis, live voice sessions, shadowing drills, quotas, background jobs, and an admin console.

Short version

We built YouChannel because authentic video is one of the best language-learning inputs, but it is usually trapped in passive watching. We stopped because the learning loop was promising, while the operating model around model cost, platform dependency, quota management, and real-time voice quality was too heavy for the stage of the project.

The product thesis

The starting belief was simple: learners do not only need more lessons. They need better ways to practice with material they already want to watch.

Traditional learning products often begin with curriculum. YouChannel began with source material: a YouTube playlist, a selected video, transcript or model-derived understanding, an AI partner, and a practice loop.

What we built

1. Library and import

YouTube OAuth connected a user's account and normalized video metadata.

2. Video analysis

A jobs service queued Gemini analysis for summaries, scenes, speakers, timestamps, and transcripts.

3. Live practice

Gemini Live powered real-time speaking sessions with learner memory and session resumption.

4. Shadowing and progress

Recent sessions produced drills, score history, and repeatable speaking practice.

Why we built it

YouTube already has the material learners want. AI can convert passive media into active context. Live voice makes practice feel less theoretical. The strongest signal was the combined loop: watch something real, ask about it, talk through it, repeat useful sentences, then see progress.

The good signal

The strongest signal was not any single feature. It was the combined loop: watch something real, ask about it, talk through it, repeat useful sentences, then see progress.

What became hard

Question What we learned Decision
Can video analysis be cheap enough? Long videos and repeated analysis introduce real model cost and latency. A quota system became necessary early.
Can YouTube be treated as stable infrastructure? OAuth, API quota, private videos, deleted videos, and policy boundaries all matter. The product needed platform-risk handling, not just import code.
Can live voice be consistently good? Real-time practice is sensitive to model behavior, audio quality, session state, and user expectations. The live loop needed serious product tuning.
Can one small project operate all of this? The app wanted frontend polish, backend jobs, admin tooling, support, billing, and content safety. The surface area was too large for the intended stage.

The quota problem was not just billing

A language-learning product has to feel generous. A video-analysis product has to control cost. Those two forces fight each other. YouChannel added quota grants, usage events, cache refresh helpers, and admin views because the system needed to know what each learner could do and when to refund a failed analysis.

The platform-dependency problem

YouChannel depended on YouTube for content access and Gemini for video understanding and live voice. Neither dependency was wrong. Together, they meant the core experience was bounded by API policy, model availability, cost, latency, and behavior that the product did not fully control.

Why we stopped

We stopped because continuing would have required turning YouChannel into an operations-heavy product before the learning wedge was narrow enough. The right next version would not have been "add more features." It would have been a sharper product: BYOK local study, shadowing-only practice, a developer analysis API, or a live speaking companion without YouTube import.

The actual conclusion

The idea was not worthless. The product was over-composed. It tried to combine content import, AI analysis, live voice, assessment, quotas, and admin operations before one wedge had earned the right to carry the rest.

Timeline

  1. Prototype
    Turn videos into structured learning data.

    Import YouTube metadata and analyze videos with Gemini.

  2. Practice
    Move from reading to speaking.

    Add live voice, personalization, assessments, and shadowing drills.

  3. Operate
    Build the machinery around the experience.

    Add quotas, jobs, admin screens, API docs, and deployment notes.

  4. Archive
    Stop before the product story became dishonest.

    Publish the code and write down the lessons instead of pretending it was launch-ready.

Closing note

YouChannel is now best understood as an open-source product archive: a real build, a useful system, and a clear stopping point. That is not a failure mode. It is one of the healthier endings for a product experiment.
